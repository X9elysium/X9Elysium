# Sample Data — Schema Reference

9 CSVs + 1 sample invoice doc, all UTF-8, comma-delimited, ISO-8601 dates. Generated by `generate_sample_data.py` (deterministic, seeded). Re-run that script any time to regenerate.

Every dataset's schema below is **stable** — when these CSVs get swapped out for live Google Sheets feeds, the column names and types stay the same so the Tableau workbook keeps working without rewiring.

---

## `leads.csv` — feeds Tab 1 (Leads & Pipeline)

One row per inbound lead.

| Column | Type | Notes |
|---|---|---|
| `lead_id` | string | `L-1001` style. Primary key. |
| `submitted_at` | datetime | ISO-8601, when the form was submitted. |
| `name` | string | Lead's full name. |
| `email` | string | Lead's email (synthetic). |
| `company` | string | Company name. |
| `phone` | string | E.164-ish. May be empty (~65% are). |
| `country` | string | Lead's country. |
| `source` | string | One of: `Organic Search`, `LinkedIn`, `Referral`, `Direct`, `Instagram`, `Email`, `Paid Search`. |
| `topic` | string | What they're asking about (free-form, but drawn from a fixed list). |
| `project_type` | string | Closest matching service category. |
| `budget_range` | string | `$X,XXX - $YY,YYY`. Treat as text/dim, not numeric. |
| `status` | string | `New`, `Qualified`, `Discovery Booked`, `Proposal Sent`, `Negotiation`, `Won`, `Lost`. **This is your funnel ordering.** |
| `lead_score` | int (0-100) | Quality score. |
| `assigned_to` | string | `Darshan Patel` or `Adhvait Patel`. |
| `first_response_at` | datetime | ISO-8601, when we first replied. Use this minus `submitted_at` for response-time metric. |
| `deal_value_usd` | int (or empty) | USD. Empty for `New`/`Qualified`. Filled for `Proposal Sent` onward. |
| `won_lost` | string | `Won`, `Lost`, or empty. |
| `lost_reason` | string | Only filled when `won_lost = Lost`. |
| `notes` | string | Free-form. Often empty. |

**Useful calc fields for the workbook:**
- `Response Time (hours)` = `first_response_at` − `submitted_at`
- `Days In Stage` = today − `submitted_at` (for non-final statuses)
- `Win Rate` = COUNT(won)/COUNT(won_lost ≠ "")
- Funnel ordering: use a CASE WHEN on `status` to assign a numeric rank.

## `seo-rankings.csv` — feeds Tab 2 (SEO Performance)

Weekly position snapshots, one row per (keyword × date).

| Column | Type | Notes |
|---|---|---|
| `date` | date | ISO-8601, the snapshot date (Sunday). |
| `keyword` | string | The search query. |
| `page_url` | string | The URL we want ranking for this keyword. |
| `position` | float | Rank position. Lower = better. 1 is the top result. 100 = not found in top 100. |
| `search_volume` | int | Estimated monthly searches. |
| `difficulty` | int (0-100) | Keyword difficulty score. |
| `search_engine` | string | Always `Google` in sample. |
| `country` | string | Search country. |
| `intent` | string | `commercial`, `informational`, `comparison`, `transactional`, `navigational`. |
| `position_change` | float (or empty) | Change vs prior snapshot. Negative = worse. Empty for the first snapshot. |

**Useful chart:** keyword on Y axis, date on X axis, color encoded by `position` (green low, red high).

## `core-web-vitals.csv` — feeds Tab 3 (Website Health)

Weekly CWV measurement per (URL × device). Use the `rating` column for the colored badge.

| Column | Type | Notes |
|---|---|---|
| `date` | date | ISO-8601, Sunday. |
| `page_url` | string | One of the 10 most-visited pages. |
| `device` | string | `mobile` or `desktop`. |
| `lcp_ms` | int | Largest Contentful Paint. < 2500 = Good. |
| `inp_ms` | int | Interaction to Next Paint (replaced FID in 2024). < 200 = Good. |
| `cls` | float | Cumulative Layout Shift. < 0.1 = Good. |
| `fcp_ms` | int | First Contentful Paint. |
| `ttfb_ms` | int | Time to First Byte. |
| `rating` | string | Pre-computed: `Good`, `Needs Improvement`, `Poor`. Use this for the traffic-light viz. |

**Thresholds (Google CWV, 2026):**
- LCP: Good ≤ 2500ms, Needs Improvement ≤ 4000ms, else Poor
- INP: Good ≤ 200ms, Needs Improvement ≤ 500ms, else Poor
- CLS: Good ≤ 0.10, Needs Improvement ≤ 0.25, else Poor

## `tasks.csv` — feeds Tab 4 (Tasks & Operations)

One row per task.

| Column | Type | Notes |
|---|---|---|
| `task_id` | string | `T-4001` style. Primary key. |
| `title` | string | Short task title. |
| `project` | string | `<Client> — <Project>` or `Internal — <area>`. |
| `client` | string | Client name (or `Internal`). |
| `assignee` | string | `Darshan Patel` or `Adhvait Patel`. |
| `status` | string | `Backlog`, `Todo`, `In Progress`, `Blocked`, `Done`. |
| `priority` | string | `High`, `Medium`, `Low`. |
| `category` | string | `Build`, `Design`, `Audit`, `Migration`, `Bug Fix`, `Strategy`, `Content`, `Sales`, `Admin`. |
| `created_at` | date | ISO-8601. |
| `due_date` | date | ISO-8601. |
| `completed_at` | date (or empty) | Filled only when `status = Done`. |
| `estimated_hours` | int | Hours estimated. |
| `actual_hours` | float (or empty) | Hours logged (filled for `Done` only). |

**Useful calcs:**
- `Is Overdue` = `due_date < today AND status ≠ "Done"`
- `Cycle Time (days)` = `completed_at − created_at` (when Done)
- `Estimate Variance` = `actual_hours − estimated_hours`

## `invoices.csv` — feeds Tab 5 (Finance & Operations)

One row per invoice. See `sample-invoice.md` for the rendered template.

| Column | Type | Notes |
|---|---|---|
| `invoice_id` | string | `INV-2026-0001` style. Primary key. Sequential within the year. |
| `client_name` | string | The client being billed. |
| `client_email` | string | AP / billing contact email. |
| `project` | string | Free-form project description. |
| `issue_date` | date | ISO-8601, the date on the invoice. |
| `due_date` | date | ISO-8601, derived from `issue_date + payment_terms`. |
| `payment_terms` | string | `Due on Receipt`, `Net 15`, `Net 30`, `Net 60`. |
| `currency` | string | `USD` or `CAD`. |
| `subtotal` | float | Pre-tax amount, in `currency`. |
| `tax_rate` | float (0-1) | Decimal. `0.13` = HST Ontario. `0` for USD invoices to non-Canadian clients. |
| `tax_amount` | float | `subtotal × tax_rate`. |
| `total` | float | `subtotal + tax_amount`. **Use this for revenue calcs.** |
| `status` | string | `Draft`, `Sent`, `Viewed`, `Partial`, `Paid`, `Overdue`, `Void`. Drives the funnel. |
| `paid_date` | date (or empty) | Filled when `status = Paid` or `Partial`. |
| `paid_amount` | float (or empty) | Equals `total` for `Paid`, less for `Partial`. |
| `payment_method` | string (or empty) | `Wise`, `Stripe`, `Bank Transfer`, etc. |
| `notes` | string | Free-form. |

**Useful calcs:**
- `Days Outstanding` = today − `issue_date` (for non-Paid)
- `AR Aging Bucket`: 0–30 / 31–60 / 61–90 / 90+ days from `issue_date` for unpaid.
- `Collection Rate` = SUM(paid_amount) / SUM(total) for the period.
- `Days Sales Outstanding (DSO)` = AVG(`paid_date` − `issue_date`) for `Paid` invoices in window.
- `Currency-normalized total`: convert CAD → USD at a fixed rate (configure in workbook parameter).

## `expenses.csv` — feeds Tab 5 (Finance & Operations)

One row per expense.

| Column | Type | Notes |
|---|---|---|
| `expense_id` | string | `EXP-0001` style. Primary key. |
| `date` | date | ISO-8601, expense date. |
| `vendor` | string | Who you paid. |
| `category` | string | `Software/SaaS`, `Hosting`, `Marketing`, `Travel`, `Meals`, `Office`, `Contractor`, `Education`, `Banking`, `Other`. |
| `subcategory` | string | Free-form sub-tag for finer rollups (e.g. `AI`, `Conference`, `Design`). |
| `amount_usd` | float | Already normalized to USD. |
| `currency` | string | Original currency before normalization. |
| `payment_method` | string | `Amex Business`, `Visa Business`, `Bank Transfer`, `PayPal`. |
| `recurring` | string | `Yes` (subscription) or `No` (one-off). |
| `billable_to_client` | string | `Yes` or `No`. If `Yes`, should be re-billed via invoice line item. |
| `client` | string (or empty) | Filled when `billable_to_client = Yes`. |
| `receipt_url` | string | Google Drive link to receipt. |
| `notes` | string | Free-form. |

**Useful calcs:**
- `Monthly Recurring Cost` = SUM(amount_usd) WHERE recurring = Yes, grouped by month.
- `Burn Rate` = SUM(amount_usd) per month.
- `Unbilled Reimbursables` = SUM(amount_usd) WHERE billable_to_client = Yes AND no matching invoice line.
- `Net Cash Flow` = SUM(invoices.total WHERE Paid) − SUM(expenses.amount_usd) per month.

## Joining hint for the workbook

If you want a single multi-source view, the only natural join key across CSVs is **`date`**. Most tabs work fine with one CSV each — only consider joins for cross-cutting questions like *"which marketing campaign drove the leads that converted"* (campaigns ↔ leads via approximate date + source).

For the v1 workbook keep each tab on a single source — simpler, fewer surprises.
